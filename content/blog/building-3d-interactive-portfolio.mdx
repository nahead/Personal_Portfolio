### 4. Responsive Design

// Device tier detection
export function detectDeviceTier() {
  const gpu = navigator.gpu;
  const memory = (navigator as any).deviceMemory;
  const cores = navigator.hardwareConcurrency;
  
  if (cores >= 8 && memory >= 8) return 'high';
  if (cores >= 4 && memory >= 4) return 'medium';
  return 'low';
}

## Challenges & Solutions

### Challenge 1: Performance on Mobile

**Problem:** Initial version ran at 20-30fps on mobile devices

**Solution:**
- Reduced particle count on mobile (1000 → 300)
- Disabled shadows on low-tier devices
- Used lower resolution textures
- Implemented LOD (Level of Detail) system

```typescript
const particleCount = tier === 'high' ? 1000 : tier === 'medium' ? 500 : 300;
const enableShadows = tier === 'high';
```

### Challenge 2: Bundle Size

**Problem:** Initial bundle was 2.5MB (too large)

**Solution:**
- Code splitting with dynamic imports
- Tree-shaking unused Three.js modules
- Lazy loading 3D components
- Result: **35% reduction** to 1.6MB

### Challenge 3: Scroll Synchronization

**Problem:** Scroll position didn't match camera position smoothly

**Solution:**
- Implemented custom scroll progress hook
- Used GSAP for smooth interpolation
- Added easing functions for natural movement

```typescript
export function useScrollProgress() {
  const [progress, setProgress] = useState(0);
  
  useEffect(() => {
    const handleScroll = () => {
      const scrolled = window.scrollY;
      const maxScroll = document.body.scrollHeight - window.innerHeight;
      setProgress(scrolled / maxScroll);
    };
    
    window.addEventListener('scroll', handleScroll);
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);
  
  return progress;
}
```

### Challenge 4: Memory Leaks

**Problem:** Memory usage increased over time

**Solution:**
- Proper cleanup in useEffect hooks
- Disposed Three.js geometries and materials
- Removed event listeners on unmount

```typescript
useEffect(() => {
  // Setup
  const geometry = new THREE.BoxGeometry();
  const material = new THREE.MeshStandardMaterial();
  
  return () => {
    // Cleanup
    geometry.dispose();
    material.dispose();
  };
}, []);
```

## Lessons Learned

### 1. Start Simple, Add Complexity

I initially tried to build everything at once. Bad idea. Start with basic 3D scene, then add features incrementally.

### 2. Performance First, Features Second

A smooth 60fps experience with fewer features beats a laggy experience with many features.

### 3. Always Provide Fallbacks

Not everyone has a powerful GPU. The 2D fallback ensures everyone can view my portfolio.

### 4. Test on Real Devices

Emulators lie. Test on actual phones, tablets, and low-end laptops.

### 5. Monitor Bundle Size

Every KB matters. Use webpack-bundle-analyzer to track what's bloating your bundle.
