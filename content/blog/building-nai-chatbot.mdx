# Building NAI: My Journey Creating an AI Chatbot

When I started building my portfolio, I wanted something that would stand out. Not just another static website, but an **interactive experience** that showcases my AI development skills. That's how NAI (Nahead AI) was born.

## The Problem

Visitors to portfolios often have questions:
- What technologies does this developer know?
- What projects have they worked on?
- Are they available for work?
- How can I contact them?

Instead of making visitors hunt for information, why not let them **ask an AI**?

## The Solution: NAI

NAI is an intelligent chatbot that:
- Answers questions about my skills, projects, and experience
- Provides context-aware responses
- Detects visitor location and time for personalized greetings
- Offers quick reply buttons for common questions
- Works seamlessly on mobile and desktop

## Tech Stack

### Backend: Python + FastAPI

```python
from fastapi import FastAPI
from openai_agents import Agent

app = FastAPI()

@app.post("/chat")
async def chat(message: str):
    response = await agent.run(message)
    return {"response": response}
```

**Why FastAPI?**
- Lightning-fast async performance
- Automatic API documentation
- Type safety with Pydantic
- Easy deployment

### AI: OpenAI Agents SDK + Gemini API

The magic happens with the **OpenAI Agents SDK**:

```python
agent = Agent(
    name="NAI",
    model="gemini-1.5-flash",
    instructions="""
    You are NAI, Nahead Jokhio's AI assistant.
    Help visitors learn about Nahead's skills,
    projects, and experience.
    """
)
```

**Why Gemini?**
- Cost-effective for high-volume usage
- Fast response times
- Good context understanding
- Free tier for development

### Frontend: Next.js + TypeScript

The chat interface is built with:
- **Framer Motion** for smooth animations
- **Zustand** for state management

## Challenges & Solutions

### Challenge 1: Response Time

**Problem:** Initial responses took 3-5 seconds

**Solution:**
- Switched from GPT-4 to Gemini Flash (faster)
- Implemented streaming responses
- Added loading states with animations

### Challenge 2: Context Management

**Problem:** Agent forgot previous conversation context

**Solution:**
- Implemented conversation history
- Limited context window to last 10 messages
- Added memory system for important facts

### Challenge 3: Rate Limiting

**Problem:** Free tier API limits

**Solution:**
- Implemented request caching
- Added rate limiting on frontend
- Optimized prompts to reduce token usage

## Results

## Lessons Learned

### 1. Start Simple

I initially tried to build a complex multi-agent system. Bad idea. Start with a single agent and add features incrementally.

### 2. User Experience > Features

A fast, simple chatbot is better than a slow, feature-rich one. Focus on response time and UX first.

### 3. Test with Real Users

My friends and family tested NAI and found bugs I never noticed. Real user testing is invaluable.

### 4. Monitor Everything

I added logging for:
- Response times
- Error rates
- Popular questions
- User satisfaction

This data helps me improve NAI continuously.

## Try It Yourself

Want to build your own AI chatbot? Here's my advice:

1. **Start with OpenAI Agents SDK** - easiest way to get started
2. **Use FastAPI** for backend - simple and fast
3. **Deploy on Render** - free tier works great
4. **Test extensively** - AI can be unpredictable
5. **Monitor costs** - API calls add up quickly
